# AI-Assisted Development Disclosure

This project was designed by the project owner and implemented with AI coding assistance.

The goal of this document is to be transparent about the development process. LootLoop is a protocol and product design exercise, not a claim that every line was manually typed by the project owner.

## Human Responsibilities

The project owner directed the work and made product/protocol decisions, including:

- Product logic.
- Protocol design.
- State machine decisions.
- Fund flow rules.
- Security boundary definition.
- Prompt engineering.
- Reviewing execution reports.
- Deciding iteration direction.
- Prioritizing which risks to fix.
- Choosing when to break compatibility during v0.3 redesign.

## AI Responsibilities

AI coding assistance was used for:

- Code generation.
- Test generation.
- Frontend implementation.
- Documentation drafts.
- Refactoring support.
- Execution report summaries.

## Why This Matters

AI-assisted protocol development still requires human judgment.

The hard parts are not only syntax. They include:

- Defining invariants.
- Deciding trust boundaries.
- Choosing account models.
- Handling funds safely.
- Understanding state transitions.
- Evaluating security tradeoffs.
- Reviewing tests and failure modes.

AI can accelerate implementation, but it does not remove the need for protocol ownership and critical review.
